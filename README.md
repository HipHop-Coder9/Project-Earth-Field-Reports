# 🌍 Project Earth: Field Reports

An environmental detective game where every mystery is built on real, live data.

Players investigate ten environmental cases across ten real US locations — from a fish die-off in Lake Erie to coral bleaching in Key Largo — by gathering evidence, playing arcade-style mini-games, and identifying the combination of human and natural causes behind each event. Every solved case ends with a prevention guide explaining how the problem could be reduced in the future.

Built for the *Put the Science in Computer Science* hackathon by Madhu and Riya.

---

## Why we built it

Climate education usually follows the same pattern: here's the problem, here's why it happened, here's what to do about it. Students absorb information instead of investigating it.

We wanted to flip that. In Project Earth, the player is the scientist. They read a case, gather their own evidence, weigh it against possible explanations, and come to a conclusion that the game checks against the actual evidence they've collected. The right answer is almost never one cause — it's always a combination of human activity and a natural condition working together. That mirrors how real environmental problems unfold.

---

## How to play

1. **Pick a case** from the sidebar. Each case is a real environmental crisis at a real US location.
2. **Read the situation** in the main panel and watch the live happenings feed for context.
3. **Check the live data** in the telemetry chart — temperature, precipitation, air quality, depending on the case.
4. **Click a clue card** to play a mini-game. Win the mini-game to unlock the clue and add it to your dossier.
5. **Pick the cause** from the suspect list once you have enough evidence.
6. **Read the prevention guide** to learn how the problem could be reduced in the future.

You manage three resources as you play:

- **Research Budget** — spent on each clue
- **Reputation** — drops when you guess wrong, climbs when you solve cases
- **Breakthrough Meter** — fills as you collect evidence

If you run low on budget, you can request additional research funding once per case at a small reputation cost.

### Mini-games (5 types)

Each clue is unlocked by a different short challenge:

- **Spot the Difference** — find what changed between a healthy environment and a polluted one
- **Match Flip** — match pairs of cards
- **Tap Rush** — react quickly to the right symbols
- **Light Show** — repeat a flashing pattern
- **Anagram Burst** — unscramble an environmental science word like `ALGAE`, `OZONE`, or `DROUGHT`

The mini-game type a clue uses rotates by case, and difficulty scales with how deep into a case you go.

## How it's built

The project runs as three files: `index.html`, `styles.css`, and `app.js`. There are no frameworks or libraries.

A few notes on specific pieces:

- **Live data** comes from [Open-Meteo](https://open-meteo.com/), a free public weather service. Weather cases pull temperature, precipitation, and wind from the forecast endpoint. Air-pollution cases pull AQI, PM2.5, and ozone from the air-quality endpoint. Each case has its own coordinates, so the data reflects current conditions at that real location.
- **Charts** are drawn directly on an HTML5 `<canvas>` element. The rendering code (axes, gridlines, multi-series lines, legend) is written from scratch rather than using a charting library.
- **Artwork** is generated by code. Every spot-the-difference scene is built as inline SVG from a small set of rules, so there are no image files for any of the in-game scenes.
- **Onboarding tour** highlights and explains each panel on first launch. It's built from a list of anchor points and a spotlight overlay.

## Running it

The game is fully static, so you can run it a few different ways.

### Option 1: open the HTML file directly

```bash
git clone <this-repo>
cd project-earth
open index.html   # macOS
# or just double-click index.html on any OS
```

This works for everything except the live API calls, which some browsers block when running from `file://`.

### Option 2: serve it locally (recommended)

If you have Python:

```bash
python3 -m http.server 8000
```

Then open `http://localhost:8000` in your browser. The live data feeds will work normally.

If you have Node.js:

```bash
npx serve
```

### Option 3: deploy it

Drop the three files onto any static host — GitHub Pages, Netlify, Vercel, or your own web server. No build step required.

## Project structure

```
project-earth/
├── index.html       # markup, panel layout, templates
├── styles.css       # all styling, including the panel grid
├── app.js           # game logic, cases, mini-games, live data, chart rendering
├── logo.jpg         # detective-Earth mascot
└── README.md        # this file
```

The `gameCases` array at the top of `app.js` holds all ten cases — adding a new one is mostly a matter of adding another entry to that list with the case details, clues, suspects, and live-data coordinates.
